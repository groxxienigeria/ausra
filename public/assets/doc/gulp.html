Gulp

A brief explanation of the gulpfile provided with Grayic's themes.

Build, Serve and Watch: gulp

Running gulp in the terminal or command line will compile the sources to dist then start a local Browsersync instance on port 4001 to serve and refresh your pages as you edit.

The default task runs the following operations:

  • clean: Gulp deletes the dist folder. All files in dist are erased.
  • html: All .html files are copied and .njk files are compiled from app to dist.
  • scss: Compiles main.css and settings.css from app/scss/ to the dist/css along with appropriate .map files.
  • copy-assets: Copies static files or assets from app to dist directory, such as fonts, images, plugins, javascript.
  • browserReload: Starts a Browsersync instance on port 4100 served from dist, defaults to index.html.
  • watchFiles: Monitors the following files and runs the corresponding tasks when anything is added, modified or deleted:
    • app/*.html and app/*.njk, then runs the html task.
    • app/scss/*.scss, then runs the scss task.
    • app/*.*: then runs the copy-assets task.
    • After copying or replacing all new or modified assets or deleting the deleted ones, gulp runs the browserReload task.
    • Continues watching for changes.

Special Feature (Plugin bundling and minification)

Plugin minification and bundling decreases server request time and potentially improves page load times a.k.a page speed.

You can use this special feature by simply running the following command on the command window or on your code terminal in the root directory of your theme folder after you have finished your customization and ready to upload your theme on your server.

gulp pluginminify

This command will create a directory named "bundled" in the root directory with two nested folders inside:

  1. css

  2. js

CSS will contain the minified bundled file named "plugins.min.css" consisting of all plugin's CSS files of app/plugins/ directory.

JS will contain the minified bundled file named "plugins.min.js" consisting of all plugin's JS files of app/plugins/ directory.

Final Step:

  1. Move the plugins.min.css to your preferred folder and link this file Replacing all plugin's css linking right before main.css linking.

  2. Move the plugins.min.js to your preferred folder and link this file Replacing all plugin's js linking right before custom.js linking.

N.B: If you are using nunjucks, then link the files in template.njk of app/partials/template.njk
